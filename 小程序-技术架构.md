# 心镜 MindMirror 微信小程序 - 技术架构文档

## 1. 架构设计

微信小程序原生开发，纯前端应用，数据本地存储。

```mermaid
graph TD
    A[小程序框架层] --> B[页面层]
    A --> C[逻辑层]
    A --> D[数据层]
    
    B --> B1[首页]
    B --> B2[测评库]
    B --> B3[测评详情]
    B --> B4[答题页]
    B --> B5[结果页]
    B --> B6[历史记录]
    B --> B7[我的]
    
    C --> C1[测评数据管理]
    C --> C2[答题状态管理]
    C --> C3[计算器引擎]
    C --> C4[工具函数]
    
    D --> D1[本地存储 Storage]
    D --> D2[内存状态]
```

## 2. 技术选型

| 技术 | 说明 |
|------|------|
| 框架 | 微信小程序原生 (WXML + WXSS + JS) |
| 基础库 | 2.30.0+ |
| 开发工具 | 微信开发者工具 (最新版) |
| 数据存储 | wx.setStorage / wx.getStorage |
| 图表 | 自定义 Canvas 或简单 CSS 图表（免费版） |

**免费版限定**：
- 不使用云开发
- 不使用第三方付费服务
- 所有逻辑前端实现
- 数据本地存储

## 6. 核心功能实现

### 6.1 测评数据管理
- 所有测评数据硬编码在 data/assessments/ 目录
- 按原有仓库的测评数据结构移植
- 支持分类、搜索查找

### 6.2 答题流程
1. 用户点击开始测评 → 进入答题页
2. 逐题作答，答案保存在内存
3. 支持上一题/下一题
4. 完成后调用计算器引擎计算结果
5. 保存到本地存储
6. 跳转到结果页

### 6.3 计算器引擎
- 移植原有仓库的 BaseProfessionalCalculator
- 实现维度计算、得分标准化、结果生成
- 纯前端计算，无后端依赖

### 6.4 数据持久化
- 使用 wx.setStorageSync 同步存储
- 页面加载时读取历史记录
- 结果自动保存，支持随时查看
