# HumanOS 部署包

## 📦 部署包说明

这是 HumanOS 专业测评平台的完整部署包，包含所有可以直接部署到网页服务器的文件。

## 🚀 部署方法

### 方法1：直接上传到服务器

1. 将 `HumanOS-deploy` 文件夹中的所有文件上传到您的网站服务器根目录
2. 确保服务器支持静态文件托管
3. 访问您的域名即可查看网站

### 方法2：使用FTP/SFTP上传

1. 使用FTP客户端（如FileZilla）连接到您的服务器
2. 将 `HumanOS-deploy` 文件夹中的所有文件上传到网站根目录
3. 确保文件权限正确（通常为644）

### 方法3：使用控制面板

1. 登录您的网站控制面板（如cPanel）
2. 进入文件管理器
3. 上传并解压 `HumanOS-deploy` 文件夹到网站根目录

## 📁 文件结构

```
HumanOS-deploy/
├── assets/                    # 静态资源文件
│   ├── *.js                   # JavaScript文件
│   └── *.css                  # 样式文件
├── 404.html                   # 404错误页面
├── CNAME                      # 域名配置文件
├── index.html                 # 主页面
└── vite.svg                   # 网站图标
```

## ⚙️ 服务器要求

- **支持静态文件托管**（HTML, CSS, JavaScript）
- **支持HTTPS**（推荐）
- **支持自定义域名**
- **无需数据库或后端服务**

## 🌐 域名配置

### 自定义域名

如果您使用自定义域名，请：

1. 修改 `CNAME` 文件，将内容改为您的域名
2. 在域名服务商处配置DNS解析
3. 等待DNS生效（通常需要几分钟到几小时）

### 默认域名

如果使用服务器默认域名，可以删除 `CNAME` 文件。

## 🔧 常见问题

### Q: 页面显示空白？

**A:** 检查以下几点：
- 确保所有文件都已上传
- 检查浏览器控制台是否有错误
- 清除浏览器缓存后重试

### Q: 样式显示异常？

**A:** 
- 确保 `assets` 文件夹完整上传
- 检查文件路径是否正确
- 清除浏览器缓存

### Q: 无法访问某些页面？

**A:** 
- 确保服务器支持SPA（单页应用）路由
- 检查 `404.html` 文件是否正确配置
